# Security Policy

## Supported versions

tracebloom is pre-alpha. No published release is supported yet. Once v0.1.0 ships (Phase 2 milestone), this section will list supported versions.

## Reporting a vulnerability

Please **do not** open a public issue for security vulnerabilities.

While the project is pre-alpha, report suspected vulnerabilities via GitHub's private vulnerability reporting: https://github.com/tracebloom/tracebloom/security/advisories/new.

We aim to acknowledge reports within 72 hours and to provide a fix or mitigation plan within 14 days for confirmed, in-scope issues.

## Scope

In scope:
- The `tracebloom` Python package (SDK + CLI)
- The `@tracebloom/dashboard` web dashboard
- The single-file HTML viewer build
- CI/CD workflows in this repository

Out of scope:
- Third-party dependencies — report those upstream
- Demo or example code that is clearly marked as illustrative

## Disclosure

We follow coordinated disclosure. A credit line in the release notes is offered for every confirmed report unless the reporter prefers to remain anonymous.
